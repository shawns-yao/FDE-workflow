# 06-MR评审智能体

## 1. 目标

在 GitLab MR 阶段使用 FDE Agent Runtime 的 `code_runtime` 进行代码和配置前置评审。高风险问题阻断合入，普通问题以 MR 评论和报告 artifact 呈现。

## 2. 不做什么

```text
不自动修改 MR 代码
不直接合并 MR
不执行生产命令
不读取仓库外目录
```

## 3. 上游输入

```text
gitlab.mr.created
gitlab.mr.updated
GitLab CI merge_request_event
评论触发 @claude review（可选）
```

## 4. 输出结果

```text
mr.review.completed
mr-review-report.md
mr-review-report.json
GitLab MR comment
阻断标签（高风险时）
```

## 5. 接口契约

### 5.1 Agent task

```json
{
  "agent_type": "pipeline",
  "business_task_type": "mr_review",
  "runtime_capability": "code_task",
  "runtime_type": "code_runtime",
  "prompt_ref": "prompts/pipeline/mr-review.md",
  "schema_ref": "schemas/agents/mr-review-result.schema.json",
  "permission_profile": "ci-readonly"
}
```

### 5.2 结果 JSON

```json
{
  "status": "passed",
  "risk_level": "medium",
  "findings": [
    {
      "category": "security",
      "severity": "high",
      "file": "src/config.ts",
      "summary": "疑似硬编码密钥",
      "recommendation": "改为环境变量注入",
      "blocking": true
    }
  ]
}
```

## 6. 处理流程

```text
1. GitLab CI 拉取 MR diff
2. 组装只读 Agent task
3. Agent Runtime 使用 `code_runtime`
4. 校验输出 JSON
5. 写报告 artifact
6. 评论到 MR
7. high / critical blocking finding 时设置失败状态
```

## 7. 文件结构

```text
agents/
  pipeline/
    mr-review/
      mr-review.md
ci/
  gitlab/
    mr-review.yml
prompts/
  pipeline/
    mr-review.md
schemas/
  agents/
    mr-review-result.schema.json
```

## 8. 与下一步衔接

通过 MR Review 后，代码进入 Pipeline Agent 确定性核心。发现配置风险但不阻断时，将风险摘要传给 `07-YAML治理智能体.md` 作为上下文。

## 9. 验证方式

```text
定向验证：用样例 diff 生成 mr-review-result JSON。
定向验证：高风险 finding 时 job 应返回失败。
不运行项目和全量测试。
```

## 10. 风险与降级

| 风险 | 降级 |
| --- | --- |
| code_runtime 不可用 | MR 评论“AI 评审未运行”，不阻断低风险环境 |
| 输出格式不合法 | 保存原始输出，标记 scan_failed |
| 误报阻断 | 允许人工移除阻断标签，保留审计报告 |
