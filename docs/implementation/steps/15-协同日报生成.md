# 15-协同日报生成

## 1. 目标

实现 Collaboration Agent 的日报能力：汇总当天构建、部署、失败、诊断、通知、进度和升级数据，生成 Markdown 报告和飞书消息。

## 2. 不做什么

```text
不替代长期 BI 平台
不依赖自建数据库作为第一版前置条件
不生成不可追溯的自然语言摘要
不把未验证推断写成确定结论
```

## 3. 上游输入

```text
gitlab.pipeline.completed
tekton.pipelinerun.completed
pipeline.build.completed
pipeline.deployment.failed
gitops.yaml.updated
argocd.application.synced
argocd.application.degraded
diagnosis.completed
collaboration.progress.updated
collaboration.escalation.triggered
manual_intervention_record（可选）
```

## 4. 输出结果

```text
collaboration.daily_report.generated
daily-report.json
daily-report.md
feishu-daily-report-card.json
```

## 5. 接口契约

### 5.1 日报结构

```json
{
  "report_id": "daily-20260617",
  "date": "2026-06-17",
  "metrics": {
    "pipeline_total": 12,
    "pipeline_failed": 2,
    "deployment_total": 8,
    "deployment_degraded": 1,
    "diagnosis_total": 3,
    "escalation_total": 1
  },
  "top_failures": [],
  "risks": [],
  "recommended_actions": [],
  "markdown_report_uri": "artifacts/reports/2026-06-17/daily-report.md"
}
```

### 5.2 报告要求

```text
必须引用 artifact 和 event id
必须区分事实、诊断结论和建议
必须标记 confidence 低的诊断
必须保留升级和未闭环事项
```

## 6. 处理流程

```text
1. 收集当天 GitLab、Tekton、GitOps、ArgoCD、diagnosis 和 collaboration 事件
2. 聚合关键指标和未闭环事项
3. 读取相关 artifact 摘要，不读取无限量日志
4. 通过 Claude API 生成日报正文
5. 校验 daily-report schema
6. 保存 Markdown 和 JSON artifact
7. 通过飞书连接器发送日报卡片
8. 输出 collaboration.daily_report.generated
```

## 8. 与下一步衔接

日报生成后进入 `16-端到端链路.md` 的闭环验证。端到端验证需要确认日报能追溯到原始事件、诊断结果、通知记录和进度状态。

## 9. 验证方式

```text
定向验证：使用一天内 mock 事件生成 daily-report.json。
定向验证：日报 Markdown 必须包含未闭环事项。
定向验证：飞书卡片发送失败时仍保存日报 artifact。
定向验证：报告内容必须引用 event_id 或 artifact_uri。
```

## 10. 风险与降级

| 风险 | 降级 |
| --- | --- |
| 当天事件不完整 | 报告中标记 partial_data |
| Claude API 不可用 | 使用规则模板生成指标型日报 |
| 飞书发送失败 | 保存报告 artifact，等待手动发送或重试 |
| 指标口径变化 | schema 增加 version，旧报告按旧版本解析 |
